# shelfsurv

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.21244011.svg)](https://doi.org/10.5281/zenodo.21244011)

A censoring-aware, disclosure-safe protocol for reconstructing month-end stock, time-on-shelf, and forward inventory depletion from unique-item resale event logs.

`shelfsurv` is the reference implementation and the reproducibility package for the MethodsX method article *"A censoring-aware protocol for reconstructing stock, time-on-shelf, and depletion from unique-item resale event logs"* (Shao and Goto, 2026). Every number, figure, and table reported in that article is regenerated by the scripts here.

## What it does

From a linked, item-level event ledger with fields `item_id`, `entry_date`, `exit_date` (nullable for items still on the shelf), and `observation_end`, the protocol produces three outputs:

1. **Exact month-end stock** by a stock-flow accounting identity. Under complete entry histories this is exact, an identity rather than an estimator.
2. **Censoring-aware time-on-shelf** via a hand-rolled Kaplan-Meier estimate with Greenwood variance and restricted mean survival time (RMST), which retains the right-censored (still-on-shelf) items instead of dropping them.
3. **Age-conditioned inventory-depletion projection** of the current stock using residual survival `S(a+h)/S(a)`, with a flat Kaplan-Meier tail as the default and a parametric tail engaged only through a non-circular held-out-calibration selection rule.

The depletion projection is characterized as an *operating envelope*: it gains at moderate right-censoring within supported horizons, and beyond that range a selection-rule parametric tail is required. A memoryless (exponential) falsification control confirms the method does not gain where item age carries no information.

No confidential data is used anywhere. All validation runs on synthetic item-level data with known ground truth.

## Requirements

Python 3.9+, with `numpy`, `pandas`, and `matplotlib`:

```
pip install -r requirements.txt
```

There is no `lifelines` or `scipy` dependency: Kaplan-Meier, Greenwood variance, RMST, and the Weibull / log-normal censored maximum-likelihood tails are implemented from first principles so the release is self-contained. Tested with Python 3.12 on Windows; Python 3.9+ is expected to work.

## Reproduce the article

```
PYTHONUTF8=1 python validate.py
```

The run is deterministic (each replication uses `seed = BASE_SEED + rep_index`; no system randomness) and writes the `figure*.png` and `table*.csv` files next to `validate.py`, overwriting the shipped copies in place. On a Japanese-locale Windows shell, `PYTHONUTF8=1` avoids code-page issues; the source and all stdout are ASCII-only. Approximate runtime: on the order of 20-30 minutes for the full 200-replications-per-cell design.

### Expected headline results (to confirm a run matches)

- **Stock honesty (E1):** reconstructed month-end stock equals the true monthly stock exactly at every censoring rate (maximum difference 0).
- **Time-on-shelf bias (E2, Fig 3):** sold-only median bias is monotone-negative (about -7.6 / -24.2 / -44.0 / -62.1 percent at 10 / 30 / 50 / 70 percent censoring); the Kaplan-Meier median stays near zero and RMST(120) within about +/-0.04 percent of the truth.
- **Depletion (E3, Fig 2):** with the flat KM tail, protocol depletion MAE about 0.0041 / 0.0412 / 0.0969 / 0.1058 vs the category-specific age-naive baseline about 0.0249 / 0.0700 / 0.0965 / 0.0868 at 10 / 30 / 50 / 70 percent censoring (protocol lower to moderate censoring, ties near 50 percent, higher at 70 percent under the short window).
- **Exponential tie (E3t, Fig 5):** under memoryless time-on-shelf the protocol does not beat age-naive.
- **Misspecification (E6, Fig 6):** no parametric tail loses to the flat tail across data-generating families; the held-out-calibration selection rule beats the flat tail in every cell (about +25-54 percent at 30-50 percent censoring across data-generating families); unsupported-query share about 0.40 to 0.56.

Small numerical differences can arise from BLAS/matplotlib versions; the qualitative pattern above should hold.

## Module summary

- **`synth.py`** : `generate(...)` simulates one long master timeline (fixed, realistic category time-on-shelf: accessory / jewelry / bag / watch medians 60 / 90 / 120 / 180 days) and induces censoring by observation-window length. `generate_scale_inflation(...)` is the demoted stress-test mechanism. Both return the ground truth (true sale dates for all in-window items, including censored ones) and the observed ledger the analyst sees.
- **`protocol.py`** : `reconstruct_stock` (Step 1), `km_dwell` / `rmst` / `km_median` (Step 2), `depletion_forecast` (Step 3), the tail options and the held-out-calibration `select_tail`, and the five practitioner baselines (`sold_only_dwell`, `censor_at_end_dwell`, `zero_opening_net_flow`, `age_naive_depletion`, `accounting_only_depletion`).
- **`validate.py`** : experiments E1 (stock honesty assert), E2 (time-on-shelf bias), E3 (depletion), E3t (exponential tie), E4 (inventory curve), E5 (stress appendix), E6 (misspecification), plus figure and table generation.

## Data availability and confidentiality

The applied motivation is a confidential second-hand-luxury reseller ledger held under a non-disclosure agreement. Those raw item-level records are **not** included in this repository and are **not required** to reproduce any result: every reported number is regenerated from the synthetic generator and the released scripts. Public, already-released aggregate monthly curves are used only as a real-scale reference for output format.
